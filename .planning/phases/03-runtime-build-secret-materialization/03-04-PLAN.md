  <action>Create `src/unraid_actuator/build_models.py` with `BuildTarget` and `BuildResult`, then create `src/unraid_actuator/build.py` with `run_build(*, runner: CommandRunner, config_path: Path = ACTIVE_CONFIG_PATH, output_root: Path | None = None) -> BuildResult`. In `run_build(...)`, load the active config, resolve `host_root = config.source_path / config.hostname`, resolve host declarations through the shared `resolve_host_apps_path(host_root)` helper from 03-01, and discover the host tree using the shared Phase 3 source contract from 03-01. Before any decryption or stage-directory creation, derive the declared target set from that resolved `apps.y[a]ml` file, map discovered environments by `(app, environment)`, and restrict build candidates to those declared targets only per BLD-01, D-10, and D-11. If any declared target is absent from discovery, or if any declared target resolves to `SourceKind.MISSING` or `SourceKind.AMBIGUOUS`, raise a hard `ValueError` immediately instead of silently skipping it or building undeclared directories. After that declared-target validation passes, resolve the final root with `build_paths.resolve_output_root`, call `build_paths.validate_output_root`, and create a sibling stage directory with `create_stage_root` before writing anything. Decrypt `secret-env.ejson` once per host through `decrypt_secret_env(...)`; for each validated declared environment in sorted `(app, environment)` order, create `stage_root / app / environment`, normalize compose from either `normalize_static_compose(...)` or `render_template_environment(...)` plus `normalize_rendered_compose(...)`, extract environment secrets with `extract_environment_secrets(...)`, merge them with any repo `.env` using `materialize_env_file(...)`, and write exactly `docker-compose.yml` plus `.env` into that output directory. After every declared environment succeeds, write the root marker file `stage_root / BUILD_MARKER_NAME` per D-10, then replace the final destination atomically with the staged tree. For the default root, overwrite any previous build tree only after the new staged tree is complete; for custom roots, rely on the earlier empty-path guard. On any failure—especially missing/ambiguous declared targets or decrypt failure per BLD-06—raise a hard error, clean up the stage directory, and leave the final output root unmodified. Add `tests/unit/test_build_service.py` that uses temporary directories and `RecordingRunner` to prove declared-only build selection, pre-decrypt failure on missing/ambiguous declared targets, shared `apps.yaml`/`apps.yml` resolution, default-root builds, marker-on-success behavior, and safe failure without partial final writes.</action>
